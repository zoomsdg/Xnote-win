# 功能规格：多标签页（Tab）+ 导入去重

> 本文是**与平台无关的产品功能规格**，描述「多标签页」与「导入去重」两项功能的行为与验收要点，供 Android / iOS / Web 等其他平台的同款产品实现参考。
> **不含任何具体平台的技术实现细节**，各端按本地技术栈落地即可。

---

## 1. 背景与目标

当前各端首页通常是一个**扁平的纪事列表**（可按分类筛选 + 搜索）。本次新增一个位于纪事之上的分区层——**标签页（Tab，又称 Notebook / 笔记本）**：

- 把纪事分组到不同标签页，像浏览器标签一样切换。
- **导入备份时可以选择导入到哪个标签页**，并自动去重，避免重复导入产生副本。

**目标收益**：多场景分区管理（如「本地 / 工作 / 从手机导入」）；重复导入同一份备份不再堆积副本。

---

## 2. 名词

| 名词 | 含义 |
|---|---|
| 标签页 / Tab | 纪事之上的分区。每条纪事**属且仅属于一个**标签页。 |
| 分类 / Category | 原有的纪事分组维度，**保持不变**。在每个标签页**内部**继续按分类筛选。 |
| 默认标签页 | 固定存在的标签页，承载所有历史纪事。可重命名，**不可删除**。 |
| 备份 ZIP | 现有的加密导出文件，跨平台互通格式。 |

> 标签页与分类是**两个独立维度**：标签页是大分区，分类是分区内的小分组。

---

## 3. 功能一：标签页

### 3.1 首页布局
- 首页顶部显示一条**标签条**，列出全部标签页；点击切换当前标签页。
- 标签条下方仍是原有的「分类筛选 + 搜索 + 纪事列表」，但**只显示当前标签页内**的纪事。
- 新建纪事时，归入**当前所在标签页**。

### 3.2 默认标签页与历史数据迁移
- 系统保证存在一个默认标签页（建议名「本地纪事」）。
- **首次升级到本版本时**：所有已有纪事自动归入默认标签页（用户无感知，数据不丢）。

### 3.3 标签页管理
- **新建**：用户输入名称即可创建一个空标签页，并切换过去。
- **重命名**：标签名可编辑（如双击标签 / 长按 / 菜单）。
- **删除**：
  - 默认标签页**不可删除**（应给出提示）。
  - 删除普通标签页时，**其下的纪事不删除，全部迁回默认标签页**；删除需二次确认。

### 3.4 持久化
- 标签页清单与「纪事 → 标签页」的归属需本地持久化（随应用数据一起，按各端既有的加密/存储方式保存）。

---

## 4. 功能二：导入到指定标签页 + 去重

### 4.1 导入流程（在原有导入基础上新增一步「选择目标标签页」）
1. 用户选择要导入的备份 ZIP。
2. **新增**：弹出「导入到哪个标签页？」选择：
   - **新建标签页**：输入新标签名，导入完成后纪事进入该新标签页；
   - **导入到现有标签页**：从下拉/列表选择一个已有标签页。
3. 输入备份密码（沿用原有密码步骤）。
4. 执行导入与去重。
5. 结束后提示结果：**新增 X 条 / 更新 Y 条 / 跳过 Z 条**。

> 体验细节：建议在**密码校验通过之后**才真正创建「新建标签页」，避免用户中途取消却残留空标签页。

### 4.2 去重规则（核心）

对备份里的每一条纪事，判断它在**目标标签页内**是否已存在：

**a) 比对范围 —— 仅目标标签页内**
- 只和**导入目标标签页**里的纪事比对。其他标签页即使有「同一条」也不算重复。
- 含义：把同一份备份导入到两个不同标签页，两边各留一份，互不影响。

**b) 判定依据 —— ID 优先，内容兜底**
1. **ID 命中**：备份纪事带有原始纪事 id；若目标标签页内某纪事的**来源 id**或**自身 id**等于它，判为同一条。
   - 为支持这条，导入新建纪事时需**记录其来源 id**（备份里那条纪事的 id），供后续再次导入比对。
   - 效果：**重复导入同一份备份 → 命中、不产生副本（幂等）**。
2. **内容兜底**：ID 未命中时，按「**标题 + 全部文本内容**」组成的内容键比对；相同则判为同一条。
   - 效果：跨设备/跨来源、id 体系不同，但**标题与正文一致**也能识别为同一条。
   - 内容键只取文本（标题 + 各文本段，按顺序拼接，去首尾空白），**不含图片/音频**。

**c) 命中后的动作 —— 按更新时间取新**
- 比较两条的**更新时间（UpdatedAt）**：
  - 导入的更**新** → **覆盖更新**目标里那条（标题、正文、媒体、分类、置顶、时间等全部以导入为准；旧媒体应清理，避免残留）。
  - 导入的更**旧或相同** → **跳过**（保留本地原样）。
- **未命中** → 作为**新纪事新增**。

**d) 同一批次内部去重**
- 同一份备份内若存在彼此重复的纪事，导入过程应让后处理的条目能识别到已处理的条目（即批内也按上述规则塌缩，不重复落库）。

### 4.3 导入与现有逻辑的关系（沿用，不改）
- 备份内的图片/音频仍按原方式还原并本地保存。
- 分类仍按原有「id 命中 → 名字命中 → 按名字新建 → 回落默认分类」解析。
- 导入的纪事在本地获得新的本地 id（与去重用的「来源 id」是两回事）。

---

## 5. 跨平台兼容性约束（重要）

- **标签页是纯本地概念，不进备份 ZIP 格式。**
  - 导出时**不写入**标签页归属字段；导入时由用户重新选择目标标签页。
  - 备份 ZIP 的 JSON 字段保持与现有跨平台格式**完全一致**，各端互导不受影响。
- **来源 id**用于本地去重，可存本地；它来自备份里纪事的 id 字段（现有格式已有），无需新增导出字段。
- 一句话：**本功能不改变任何跨平台数据格式**，只在各端本地新增「标签页」分区与「导入去重」行为。

---

## 6. 验收要点（Acceptance Criteria）

**标签页**
- [ ] 升级后出现默认标签页，历史纪事全部在其中，数据无丢失。
- [ ] 可新建、重命名标签页；切换标签页只显示该标签页内纪事。
- [ ] 删除普通标签页后，其纪事迁回默认标签页；默认标签页不可删除。
- [ ] 标签页与归属在重启后保持。

**导入去重**
- [ ] 导入时可选「新建标签页」或「现有标签页」，纪事进入所选标签页。
- [ ] **同一份备份重复导入到同一标签页 → 全部跳过，无副本。**
- [ ] 修改某条后重新导出再导入 → 该条「更新」、其余「跳过」，旧媒体无残留。
- [ ] 标题+正文相同但来源不同的纪事 → 识别为重复。
- [ ] 同一份备份导入到**另一个**标签页 → 正常新增（跨标签页不互相去重）。
- [ ] 导入结束提示「新增 / 更新 / 跳过」条数。
- [ ] 导出的备份仍可被其他平台正常导入（格式未变）。

---

## 7. 边界与异常
- 密码错误：维持原有提示，且若已为本次导入新建了空标签页，应回收，不留空壳。
- 空标题/空正文：内容键按空串处理，不应误判全部空纪事为同一条之外的异常（仍以 ID 优先）。
- 超大导入：沿用各端既有的数量/长度保护策略。

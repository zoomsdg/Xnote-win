# XNote-Win 功能改进指引

> 本文以「多标签页（Tab）+ 导入去重」这次改进为范例，沉淀本项目做功能改进时**通用的架构约束、改动套路与避坑点**，供后续同项目的功能开发参考。
>
> 配套阅读：根目录 [README.md](../README.md)（启动 / 导入 / 发布）。

## 一、先记住三条铁律

这三条是本项目的「地基」，几乎每个功能都会碰到，破坏它们会引发跨平台或数据安全问题：

1. **导出 ZIP 的 JSON 格式 = Android 端契约，不可改。**
   字段名/类型/可空性必须与 Android 端 Gson 序列化逐字段一致（见 [ExportModels.cs](../src/XNote.Core/ImportExport/ExportModels.cs) 注释）。
   → **新加的本地字段绝不能进导出 DTO `ExportNote`/`ExportBlock`**，否则两端互导会错乱。
   → 确需让新内容随导出走（如「附件」）时，**只扩展取值、不扩展字段**：新增 `type="file"` 复用已有的
   `mediaFileName`/`alt`，并把 `text` 填成 `"[附件] 原文件名"` —— 老客户端不认识该 type 会回落成文本块，
   至少看得见附件名而不是空块。这类扩展需在 Android 端补一行 type 映射才能双向可用。

2. **本地落盘一律 DPAPI 加密（`XNW1` 头），跨设备不可解。**
   `notes.json` / `categories.json` / `media/*` 都经 [MediaCryptor](../src/XNote.Core/Storage/MediaStore.cs) 加密，密钥绑定当前 Windows 账户。
   → 跨设备/跨平台传输只能走「导出加密 ZIP」（密码保护、内部媒体明文）。详见导出导入两套加密机制的区别。

3. **能力分层：Core 不依赖 WPF，App 不写存储逻辑。**
   - `XNote.Core`：模型、存储（`LocalStore`）、导入导出（`ExportImportService`）。纯逻辑、可被 `XNote.Verify` 直接测试。
   - `XNote.App`：WPF UI、对话框、媒体播放。只调用 Core 的公开方法。
   → 新功能优先把逻辑放进 Core，UI 只做编排。

## 二、本次改进做了什么（范例）

**需求**：首页扁平列表 → 标签页（Tab）布局；导入时可选「新建 tab」或「现有 tab」；tab 名可编辑；导入按规则去重。

**去重规则（与产品确认后定）**：仅目标 tab 内比对；**ID 优先**（来源 id / 本地 id）、**内容兜底**（标题+正文）；命中后**按 `UpdatedAt` 取新**（更新覆盖并清旧媒体，否则跳过）。

**落地要点**：
- 新增「标签页（Notebook）」分区层，位于纪事之上；每条纪事属且仅属一个 tab。
- `NotebookId` / `SourceId` 是**纯本地字段**（只进 `notes.json`，不进导出 ZIP）—— 严守铁律 1。
- tab 持久化于新文件 `data/tabs.json`，同样 DPAPI 加密。
- 启动时迁移：历史纪事 `NotebookId` 为空 → 默认 tab `"local"`。

提交记录：`8f670b9`（功能）、`913170b`（修 BAML 崩溃）、`75050bd`（修对话框布局）。

## 三、标准改动套路（按层）

照这个顺序改，编译错误会一路引导你补齐调用点：

### 1) 模型层 — [Models.cs](../src/XNote.Core/Models/Models.cs)
- 给 `Note` 加本地字段时，**给默认值**（如 `NotebookId = "local"`），保证反序列化老数据不炸。
- 新实体（如 `Notebook`）放这里，保持 POCO（无逻辑）。

### 2) 路径层 — [AppPaths.cs](../src/XNote.Core/Storage/AppPaths.cs)
- 新增数据文件就加一个 `XxxFile => Path.Combine(DataDir, "xxx.json")`。目录已在构造函数 `CreateDirectory`。

### 3) 存储层 — [LocalStore.cs](../src/XNote.Core/Storage/LocalStore.cs)（改动重心）
- **加载/持久化**：仿 `_categories` 加 `List<T>` 字段 + `SaveXxx()`，复用现成的 `ReadJson`/`WriteJson`（自带 DPAPI 加解密，别自己写文件 IO）。
- **种子 + 迁移**：仿 `EnsureDefaultCategories()` 写 `EnsureDefaultXxx()`；老数据迁移放 `Load()` 内，迁移发生就 `SaveNotes()` 落盘。
- **查询过滤**：给 `ListNotes(notebookId, categoryId, query)` 这类方法**追加可选参数**，保持旧调用兼容（无参 = 全量，导出全部仍用它）。
- **CRUD + 受保护项**：仿 `ProtectedCategoryIds` / `DeleteCategory`（删除时把子项迁回默认、禁止删默认）。

### 4) 导入导出 — [ExportImportService.cs](../src/XNote.Core/ImportExport/ExportImportService.cs)
- **通常不用动这一层**（它就是跨平台契约）。本次去重逻辑全部写在 `LocalStore.Import` 里，`ExportImportService` 只负责解 ZIP → `ImportNote`。
- 去重决策的可复用骨架（本次实现，未来类似需求可照抄）：
  - `FindDuplicate`：先按 ID（`SourceId`/本地 `Id` == 导出 id），再按 `ContentKey`（标题+正文）。
  - 命中 → 比 `UpdatedAt`：新则**原地更新**（先 `TryDeleteMedia` 清旧媒体再重建 blocks，**保留本地 Id**），旧则跳过。
  - 未命中 → 新建（新本地 GUID + `SourceId` 记来源）。
  - 处理一条即加入 `_notes`，**同批次内部重复会被后续条目自动识别**。
  - 返回 `record struct ImportSummary(Added, Updated, Skipped)`，UI 据此提示。

### 5) UI 层 — [MainWindow.xaml](../src/XNote.App/MainWindow.xaml) / [.cs](../src/XNote.App/MainWindow.xaml.cs)
- 复用现有命名控件（`NoteList`/`SearchBox`/`CategoryFilter`），只在 `Refresh()` 里多传一个过滤维度，改动最小。
- 对话框复用现成模式：
  - 取一个字符串 → `InputDialog.Ask(owner, title, prompt, initial)`。
  - 输密码 → `PasswordDialog.AskOnce` / `AskNew`。
  - 复杂选择（如导入目标）→ 仿 [PasswordDialog](../src/XNote.App/PasswordDialog.xaml.cs) 写静态 `Ask(...)` 返回结果对象，见 [ImportTargetWindow](../src/XNote.App/ImportTargetWindow.xaml.cs)。
- **危险动作放在确认之后**：如「新建 tab」改到密码确认之后再建，失败时 `CleanupEmptyNewTab` 回收空 tab——别让用户取消却残留垃圾数据。

### 6) 自测 — [tools/XNote.Verify/Program.cs](../tools/XNote.Verify/Program.cs)
- 改了 Core 公开签名/语义，**必须**同步更新这里，并尽量**加一段新用例**（本次加了 tab + 去重往返断言）。
- 跑：`& "C:\Program Files\dotnet\dotnet.exe" run --project tools\XNote.Verify`，要求 `== 全部通过 ✅ ==`。

## 四、WPF 避坑清单（血泪）

- **❌ 别在 `Style` / 模板里混用 `EventSetter` + `Setter.Value` 内带 `Click` 的元素。**
  会打乱 BAML 的 `connectionId` 编号，运行时 `InitializeComponent` 抛 `Unable to cast 'MenuItem' to 'Border'` 之类、**启动即崩**（本次 `913170b` 踩过）。
  ✅ 正解：`Style` 只放外观 `Setter`；双击/右键等交互在**代码后置**处理——在控件上挂 `MouseDoubleClick`/`PreviewMouseRightButtonUp`，用 `VisualTreeHelper` 回溯到容器（如 `TabItem`）取 `DataContext`，右键菜单也在代码里 new `ContextMenu`。
  > 注意：直接挂在元素上的 `ListBox.ContextMenu`（非 Style 内）是安全的，原代码一直正常。

- **❌ 固定 `Height` + 把控件塞进 `*` 行**，窗口偏小时该行被压扁、控件互相遮挡（本次 `75050bd`：现有标签页下拉被按钮盖住）。
  ✅ 正解：交互控件用独立 `Auto` 行，按钮区给 `Margin` 顶距，窗口用 `SizeToContent="Height"` 自适应。

- **改了 XAML 必须重新 `dotnet build`**：BAML 是编译产物，直接双击旧 exe 看不到改动、还可能误判。

## 五、改完自检 Checklist

- [ ] 新本地字段**没有**出现在 `ExportNote`/`ExportBlock`（铁律 1）。
- [ ] 新数据文件走 `ReadJson`/`WriteJson`，落盘是加密的（`MediaCryptor.IsEncrypted` 为真）。
- [ ] 老数据能正常加载（新字段有默认值 / `Load()` 内有迁移）。
- [ ] `XNote.Verify` 通过，且对新逻辑加了断言。
- [ ] `dotnet build XNote.sln` 0 警告 0 错误。
- [ ] 实际启动 `XNote.App.exe`，主窗口正常加载、关键交互手点一遍。
- [ ] 涉及跨平台的改动：与 Android 端导出的 ZIP 仍可互导（必要时双向验证）。

## 六、常用命令

```powershell
# 本机 dotnet 不在 PATH，用全路径
$dotnet = "C:\Program Files\dotnet\dotnet.exe"

& $dotnet build XNote.sln -nologo -v q          # 构建
& $dotnet run  --project tools\XNote.Verify     # 导入导出 + 去重自测
& $dotnet run  --project src\XNote.App          # 启动应用
```
